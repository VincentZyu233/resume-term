# Resume-Term 布局与 Session 管理四阶段方案

## Phase 1 — 启动布局显式标识

当前已有 "Set startup layout" 按钮，但无视觉反馈。目标：让用户直观知道「哪个 session 是启动布局」，并能一键切换。

### 改动
- `WorkspaceConfig`：`selectedSession`（int 索引）→ `startupSessionId`（String UUID），索引在删除 session 时会漂移，UUID 更健壮
- `WorkspaceSession`：新增 `id` 字段，序列化/反序列化包含 `id`
- `fromYamlString`：兼容旧格式 `selected_session`，自动迁移到新字段
- `WorkspaceSummary` 标题行右侧加 ⭐ 图标：
  - 若当前 session 是 startup → 实心星（amber）
  - 否则 → 空心星
  - 点击即设此 session 为 `startupSessionId` 并立即持久化
- 移除 toolbar 中旧的 "Set startup layout" 按钮（逻辑已迁移到 WorkspaceSummary）
- 配置加载时 `activeSession` getter 优先按 `startupSessionId` 查找，找不到则 fallback 到 `sessions.first`

### 完成状态
✅ 已完成（2026-06-16）

## Phase 2 — Session 切换与管理（单窗口单 Session，允许多开）

当前 UI 只能看到一个 session，无法切换。目标：应用内只管理一个 session（活跃 session），但允许多个 config 文件 / 多开实例。

> 核心思路：resume-term 窗口一次只打开一个 session（含 split 布局树），
> 但支持多开——用户复制一份可执行文件到不同目录，每个实例独立管理自己的 config.yaml。
> 这与 VS Code 的 `--user-data-dir` 思路一致。

### 改动
- `WorkspaceSummary` 顶部加 `DropdownButton<String>` 列出所有 session（以 name 显示）
- 切换 dropdown = 切换 `activeSession`，主区域树和右侧 Inspector 联动刷新
- 当前 session 名称旁显示 ⭐ 启动布局标识（来自 Phase 1）
- 底部加 "Set as startup" 按钮（与 ⭐ 功能冗余，但更显眼）
- 新增 `_switchSession(String? sessionId)` 方法

### 多开支持（不修改代码）
- 用户复制 `resume-term.exe` 到独立目录（如 `X:\packs\resume-term\resume-term-windows-x64-v0.0.1-alpha.1\`）
- 每个实例读取自己目录下的 `config/resume-term/config/config.yaml`
- 天然隔离，无需额外逻辑

## Phase 3 — Session CRUD

允许用户在 UI 中管理 session 生命周期。

### 改动
- **新建 layout**：WorkspaceSummary 底部 `+` 按钮 → 弹出命名对话框 → 创建默认单 pane session → 自动切换过去
- **重命名 layout**：WorkspaceSummary 中 session 名称双击 / 右键菜单 "Rename" → 内联编辑或弹窗
- **删除 layout**：右键菜单 "Delete" → 确认 dialog → 若删除的是 startup session，自动选第一个剩余 session 为 startup
- **复制 layout**：右键菜单 "Duplicate" → 当前 session 深拷贝（含完整布局树）→ 名称加 "(Copy)" → 追加到 `sessions` 列表

---

## Phase 4 — 导入导出 YAML

### 改动
- **导出**：toolbar 加 "Export" 按钮 → 调用 `file_selector` 弹出保存对话框 → 将 config 或当前 session 写为 `.yaml` 文件
- **导入**：toolbar 加 "Import" 按钮 → 弹出文件选择器选 `.yaml` → 解析后合并/替换到当前配置
- 导入时校验格式兼容性，失败时显示错误信息
